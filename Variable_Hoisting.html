Variable Hoisting

Hoisting is Javascript's default behaviour of moving declaration to the top of the function, if defined in a function,or top of the global context, if outside a function.

Hoisting is a javascript mechanism where variables and funtions declarations are moved to the top of their scope before code execution.
Inevitably,this means that no matter where functions and variables are declared, they are moved to the top of their scope regardless of whether their scope is global or local.

var a : Variable declaration

a = 10: Variable Initialization

If we write var a = 10; //Javascript divide this statement to two parts.

var a : Declaration

a = 10 : Initialization

Javascript moved declaration part to the top of the code if declared inside the code or top of the function code if declared inside a function.

Only Declaration part moves not Initialization.
